üíª Technical Questions

What databases does FIT-BANK support? +

FIT-BANK is database-agnostic thanks to its Hibernate persistence layer. We support all major database systems including Oracle, PostgreSQL, EnterpriseDB, SQL Server, IBM DB/2, and Sybase. This flexibility allows you to choose the database that best fits your infrastructure and budget.

Can FIT-BANK integrate with our existing systems? +

Yes! FIT-BANK provides comprehensive integration capabilities through:

  • RESTful APIs for modern system integration
  • SOAP/XML web services for legacy systems
  • Database-level integration when needed
  • Pre-built connectors for ERP systems, payment gateways, credit bureaus, SWIFT, and local clearing systems (SPI/SCI)
What is the system uptime guarantee? +

FIT-BANK is designed for 24/7/365 non-stop operation with a 99.9% uptime SLA. We provide built-in redundancy, automatic failover mechanisms, and continuous availability features to ensure your banking operations never stop.

What are the minimum server requirements? +

Minimum Requirements:

  • 16GB RAM, 4-core CPU, 500GB SSD storage
  • Linux or Windows Server operating system
  • Java 1.8 or higher runtime environment

Recommended: Requirements scale based on institution size (members, transaction volume). We support cloud deployments on AWS, Azure, Google Cloud, and on-premise infrastructure.

How does data migration work? +

Our structured 2-4 week migration process includes:

  1. Data extraction from your legacy system
  2. Data cleansing and validation
  3. Data transformation and mapping to FIT-BANK structure
  4. Trial imports and testing
  5. Final migration to production
  6. Verification and reconciliation

Our team has experience migrating from all major core banking systems in Latin America.

How do you handle regulatory changes? +

Our dedicated compliance team monitors regulatory changes in all 8 countries we serve. When regulations change:

  1. Our team analyzes the regulatory impact
  2. Updates are developed and tested
  3. Customers are notified in advance
  4. Updates are deployed with documentation
  5. Support is provided for transition

Regulatory compliance updates are included in your support package at no additional cost.

Is the system auditable? +

Yes, completely! FIT-BANK provides comprehensive audit trails:

  • Transaction logging: Every transaction recorded with full details
  • User activity tracking: User ID, timestamp, IP address, actions performed
  • Data change history: Before/after values for all modifications
  • Audit reports: Pre-built reports for internal and external auditors
  • Tamper-proof logs: Audit trails cannot be modified or deleted
What security protocols are in place? +

FIT-BANK implements multiple security layers:

  • Encryption: AES-256 for data at rest, TLS 1.3 for data in transit
  • Authentication: Multi-factor authentication, biometric support, Active Directory integration
  • Authorization: Role-based access control with granular permissions
  • Network security: Firewall configuration, IP whitelisting, VPN support
  • Testing: Annual penetration testing by third parties
  • Monitoring: Real-time fraud detection and suspicious activity alerts
Is FIT-BANK compliant with GDPR? +

Yes, FIT-BANK includes features to support GDPR compliance:

  • Data privacy controls and consent management
  • Right to erasure (right to be forgotten) functionality
  • Data portability for customer data export
  • Privacy by design architecture
  • Data breach notification capabilities

We also support compliance with local data protection regulations in Latin America.
