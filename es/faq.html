Obtén respuestas a preguntas comunes sobre FIT-BANK

💻 Preguntas Técnicas

¿Qué bases de datos soporta FIT-BANK? +

FIT-BANK es agnóstico a la base de datos gracias a su capa de persistencia Hibernate. Soportamos todos los principales sistemas de bases de datos incluyendo Oracle, PostgreSQL, EnterpriseDB, SQL Server, IBM DB/2 y Sybase. Esta flexibilidad le permite elegir la base de datos que mejor se adapte a su infraestructura y presupuesto.

¿FIT-BANK puede integrarse con nuestros sistemas existentes? +

¡Sí! FIT-BANK proporciona capacidades integrales de integración a través de:

  • APIs RESTful para integración de sistemas modernos
  • Servicios web SOAP/XML para sistemas legacy
  • Integración a nivel de base de datos cuando sea necesario
  • Conectores pre-construidos para sistemas ERP, pasarelas de pago, burós de crédito, SWIFT y sistemas de compensación locales (SPI/SCI)
¿Cuál es la garantía de tiempo de actividad del sistema? +

FIT-BANK está diseñado para operación continua 24/7/365 con un SLA de 99.9% de tiempo de actividad. Proporcionamos redundancia integrada, mecanismos automáticos de conmutación por error y características de disponibilidad continua para asegurar que sus operaciones bancarias nunca se detengan.

¿Cuáles son los requisitos mínimos del servidor? +

Requisitos Mínimos:

  • 16GB RAM, CPU de 4 núcleos, 500GB de almacenamiento SSD
  • Sistema operativo Linux o Windows Server
  • Entorno de ejecución Java 1.8 o superior

Recomendado: Los requisitos escalan según el tamaño de la institución (miembros, volumen de transacciones). Soportamos implementaciones en la nube en AWS, Azure, Google Cloud e infraestructura on-premise.

¿Cómo funciona la migración de datos? +

Nuestro proceso estructurado de migración de 2-4 semanas incluye:

  1. Extracción de datos de su sistema legacy
  2. Limpieza de datos y validación
  3. Transformación de datos y mapeo a la estructura de FIT-BANK
  4. Importaciones de prueba y testing
  5. Migración final a producción
  6. Verificación y conciliación

Nuestro equipo tiene experiencia migrando desde todos los principales sistemas bancarios centrales en América Latina.

📊 Preguntas de Negocio

¿Cuál es el tiempo típico de retorno de inversión (ROI)? +

La mayoría de las instituciones ven un ROI positivo dentro de 18-24 meses a través de:

  • Ganancias en eficiencia operativa - Reducción del 60-70% en procesamiento manual
  • Ahorro de costos - Costos reducidos de mantenimiento y soporte
  • Crecimiento de ingresos - Capacidad de lanzar nuevos productos rápidamente
  • Satisfacción del cliente - Mejor servicio que conduce a retención de miembros

Nuestros clientes han logrado un promedio de 454% de crecimiento de activos después de la implementación.

¿Proporcionan capacitación al personal? +

¡Sí! Proporcionamos programas integrales de capacitación:

  • Capacitación presencial en su institución
  • Sesiones de capacitación virtual vía videoconferencia
  • Programas de capacitación de capacitadores para construir experiencia interna

¿Podemos personalizar módulos y flujos de trabajo? +

¡Absolutamente! FIT-BANK es altamente personalizable:

  • Configuración de productos - Cree productos financieros personalizados sin programación
  • Personalización de flujos de trabajo - Ajuste procesos de aprobación y reglas de negocio

¿Qué soporte continuo proporcionan? +

Ofrecemos tres niveles de soporte:

  • Soporte Básico: Horario laboral, sistema de email/tickets, respuesta en 48 horas
  • Soporte Premium: Horario extendido, teléfono/email/chat, respuesta en 24 horas para no críticos, 4 horas para críticos
  • Soporte Enterprise: 24/7/365, gerente de cuenta dedicado, respuesta en 1 hora para problemas críticos

Todos los niveles incluyen actualizaciones de software, corrección de errores y actualizaciones de cumplimiento normativo.

¿Con qué frecuencia se publican actualizaciones? +

Nuestro calendario de lanzamientos:

  • Lanzamientos mayores: Trimestralmente (nuevas características y mejoras)
  • Lanzamientos menores: Mensualmente (mejoras y optimizaciones)
  • Parches de seguridad: Según sea necesario (implementación inmediata para problemas críticos)
  • Actualizaciones normativas: Inmediatamente tras cambios en regulaciones

Las actualizaciones se prueban exhaustivamente y pueden desplegarse con mínima interrupción.

⚖️ Preguntas de Cumplimiento y Seguridad

¿Cómo manejan los cambios normativos? +

Nuestro equipo dedicado de cumplimiento monitorea cambios normativos en los 8 países que atendemos. Cuando las regulaciones cambian:

  1. Nuestro equipo analiza el impacto normativo
  2. Se desarrollan y prueban las actualizaciones
  3. Se notifica a los clientes con anticipación
  4. Se despliegan las actualizaciones con documentación
  5. Se proporciona soporte para la transición

Las actualizaciones de cumplimiento normativo están incluidas en su paquete de soporte sin costo adicional.

¿El sistema es auditable? +

¡Sí, completamente! FIT-BANK proporciona pistas de auditoría integrales:

  • Registro de transacciones: Cada transacción registrada con detalles completos
  • Seguimiento de actividad de usuario: ID de usuario, marca de tiempo, dirección IP, acciones realizadas
  • Historial de cambios de datos: Valores antes/después para todas las modificaciones
  • Reportes de auditoría: Reportes pre-construidos para auditores internos y externos
  • Registros a prueba de manipulación: Las pistas de auditoría no pueden ser modificadas o eliminadas
¿Qué protocolos de seguridad están implementados? +

FIT-BANK implementa múltiples capas de seguridad:

  • Encriptación: AES-256 para datos en reposo, TLS 1.3 para datos en tránsito
  • Autenticación: Autenticación multifactor, soporte biométrico, integración con Active Directory
  • Autorización: Control de acceso basado en roles con permisos granulares
  • Seguridad de red: Configuración de firewall, whitelisting de IP, soporte VPN
  • Pruebas: Pruebas de penetración anuales por terceros
  • Monitoreo: Detección de fraude en tiempo real y alertas de actividad sospechosa
¿FIT-BANK cumple con GDPR? +

Sí, FIT-BANK incluye características para soportar el cumplimiento de GDPR:

  • Controles de privacidad de datos y gestión de consentimiento
  • Derecho al olvido funcionalidad de borrado
  • Portabilidad de datos para exportación de datos del cliente
  • Arquitectura de privacidad por diseño
  • Notificación de violación de datos capacidades

También soportamos el cumplimiento con regulaciones locales de protección de datos en América Latina.

💰 Preguntas de Precios e Implementación

¿Cuánto cuesta FIT-BANK? +

El precio varía según:

  • Tamaño de la institución (número de miembros/clientes)
  • Módulos seleccionados (solo paga por lo que necesita)
  • Modelo de implementación (nube vs. on-premise)
  • Nivel de soporte (Básico, Premium o Enterprise)

Ofrecemos modelos de licenciamiento flexibles incluyendo precios por usuario, por institución y basados en transacciones. Contáctenos para una cotización personalizada adaptada a las necesidades de su institución.

¿Cuál es el tiempo típico de implementación? +

Tiempo típico: 3-6 meses desde el contrato hasta el go-live, desglosado como:

  • Descubrimiento y Requisitos: 2-3 semanas
  • Configuración del Sistema: 4-6 semanas
  • Migración de Datos: 2-4 semanas
  • Pruebas y Capacitación: 3-4 semanas
  • Go-Live y Soporte: 1-2 semanas

El cronograma puede acelerarse para necesidades urgentes o extenderse para migraciones complejas.

¿Podemos comenzar con pocos módulos y agregar más después? +

¡Absolutamente! La arquitectura modular de FIT-BANK le permite:

  • Comenzar con módulos esenciales para sus operaciones
  • Agregar módulos adicionales a medida que crecen sus necesidades
  • Escalar gradualmente sin interrumpir las operaciones existentes
  • Pagar solo por los módulos que realmente utiliza

Muchos clientes comienzan con banca central y gradualmente agregan CRM, banca móvil y otros módulos.

¿Ofrecen implementación en la nube? +

¡Sí! FIT-BANK está listo para la nube y soporta:

  • Nube pública: AWS, Microsoft Azure, Google Cloud Platform
  • Nube privada: Infraestructura de nube dedicada
  • Nube híbrida: Mezcla de nube y on-premise
  • On-premise: Implementación tradicional en servidor

La implementación en la nube ofrece implementación más rápida, escalamiento automático y costos de infraestructura reducidos.
